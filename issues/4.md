## Overview
Transform the current single-page dashboard into a multi-page carousel where each page displays a list of items on the left (similar to the old column style) and detailed content for the currently highlighted item on the right. Both the pages and the highlighted items within each page should rotate automatically.

## Current State
- All dashboard columns are rendered simultaneously on a single page
- Progress bar tracks overall dashboard state
- No item-level detail view

## Desired Outcome

### Page Structure
- Each page represents what was previously a column
- **Left side:** List of items (similar to the old column layout)
- **Right side:** Expanded detail view of the currently highlighted item
- The highlighted item in the left list rotates on a timer
- The right side updates to show details of the currently highlighted item

### Rotation Behavior
- **Page-level rotation:** Pages cycle through automatically (preserve existing progress bar)
- **Item-level rotation:** Within each page, the highlighted item cycles through the list
- Two independent timers: one for page switching, one for item highlighting

### Progress Bar
- **IMPORTANT:** Preserve the current progress bar functionality
- Progress bar tracks **page-level** transitions only (not individual item rotations)
- Progress bar should reset when moving to the next page

## Acceptance Criteria
- [ ] Each page displays a list view on the left side (styled similar to old columns)
- [ ] Each page displays a detail view on the right side
- [ ] Items in the left list auto-highlight on a timer
- [ ] Right side detail view updates when highlighted item changes
- [ ] Pages auto-rotate independently of item rotation
- [ ] Existing progress bar is preserved and tracks page transitions
- [ ] Progress bar does NOT track item-level rotations
- [ ] Visual indicator showing which item is currently highlighted
- [ ] Smooth transitions between highlighted items
- [ ] Responsive design maintained

## Technical Considerations
- Two separate timers: page rotation timer and item highlight timer
- Consider making both timers configurable
- Ensure progress bar logic is decoupled from item rotation
- Determine optimal timing for item rotation vs page rotation
- Consider hover/interaction behavior (pause rotation?)
- Performance optimization for detail view rendering
